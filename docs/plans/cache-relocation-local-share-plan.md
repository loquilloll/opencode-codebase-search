# Cache Relocation to XDG Data Path Plan

## Objective

Move index hash-cache storage from workspace-local `.opencode` state into a user-local data path, with one cache file per workspace:

- `~/.local/share/opencode-codebase-search/ws-<sha256(worktree)[:16]>.cache.json`

## User-Approved Scope

- Keep one cache file per workspace hash.
- Use the exact filename format above (no per-workspace subdirectory).
- One-time migrate legacy cache from:
  - `<worktree>/.opencode/codebase-search/ws-<sha256(worktree)[:16]>.cache.json`
- Migration semantics are move (not copy).

## Guardrails

- Preserve current collection naming parity (`ws-<hash>`) and cache-key semantics.
- Do not change Qdrant storage behavior or payload schema.
- Keep migration idempotent:
  - if new cache exists, prefer new path and do not overwrite it
  - if only legacy exists, move it to new path
- Handle cross-device moves safely (`rename` fallback to `copyFile + unlink`).

## Non-Goals (for this slice)

- No change to indexing modes (`disabled`, `query`, `background`).
- No ranking or ignore behavior changes.
- No changes to provider/model settings contracts.

## Commit-Safe Phase Plan

### Phase 1 - Config Path Contract

**Goal**
Define canonical and legacy cache paths in config without changing runtime behavior yet.

**Changes**
- Update `src/tools/codebase-search/config.ts`:
  - set canonical cache root to `~/.local/share/opencode-codebase-search`
  - compute canonical `cacheFilePath` as `ws-<hash>.cache.json`
  - add helper for legacy cache path under `<worktree>/.opencode/codebase-search/`
- Update `src/tools/codebase-search/types.ts` only if additional path metadata is needed.

**Commit safety**
- Path computation only; no cache read/write migration yet.

**Validation gate**
- `npm run test:focused`

---

### Phase 2 - Legacy Move Migration in Cache Layer

**Goal**
Implement one-time legacy-to-canonical move semantics in cache handling.

**Changes**
- Update `src/tools/codebase-search/cache.ts`:
  - extend cache initialization to accept optional legacy path
  - before normal load, attempt migration when canonical file is missing and legacy file exists
  - perform move via `rename`, with `EXDEV` fallback (`copyFile` then `unlink`)
  - best-effort cleanup path when canonical already exists

**Commit safety**
- Migration stays local to cache-layer startup and does not affect indexing/search flow.

**Validation gate**
- `npm run test:focused`

**Commit safety**

- Legacy workspace cache file is moved on first eligible run.

- Focused tests pass and release verification remains green.
